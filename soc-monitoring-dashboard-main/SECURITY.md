# Security Policy

This repository is designed for authorized defensive security work, training, and portfolio demonstration.

## Supported Use

- Run scans only against systems you own or have explicit written permission to test.
- Store secrets in environment variables or a managed secret store.
- Rotate default credentials before any public deployment.
- Keep dependencies patched and review CI results before release.

## Reporting Vulnerabilities

Open a private security advisory or contact the maintainer with:

- Affected component
- Reproduction steps
- Impact summary
- Suggested mitigation

Do not publish exploit details before a fix is available.
